# bizmote コーポレートサイト 03: デザインシステム仕様書

> 本ドキュメントは、bizmote.jp のすべての視覚的判断の基準を定義する。色・タイポグラフィ・余白・コンポーネント・アニメーションすべての設計言語をここに集約する。実装時に視覚判断で迷ったら、必ず本ドキュメントに立ち戻ること。

---

## 1. デザイン哲学

### 1.1 4 つの基本原則

bizmote のデザインは、以下 4 原則に従う。

#### 原則 1: 静謐(せいひつ)

派手さではなく、静けさで品位を表現する。
余白を恐れず、要素を詰め込まない。
動きは緩やかで、瞑想的に。

#### 原則 2: 文芸性

ビジネスサイトの定型を避け、雑誌(Brutus / Casa Brutus / Pen)のような編集デザインを目指す。
タイポグラフィは情報伝達の道具ではなく、表現そのものとして扱う。

#### 原則 3: 哲学性

デザインがブランドの哲学を体現する。
「次の常識をつくる」を、サイトを訪れた瞬間に体感できるようにする。

#### 原則 4: 引き算

加えるのではなく、引いていく。
一つの要素が際立つために、他の要素を削る。
「これも入れたい」と思った時、まず「何を削れるか」を考える。

### 1.2 やらないことリスト

bizmote のデザインでは、以下を **絶対に** やらない:

- 紫グラデーション(AI 時代の汎用クリシェ)
- ガラスモーフィズム(2020 年代前半の流行)
- ステレオタイプなテック画像(回路基板、グローブ、ノードグラフ)
- ストック写真の使用
- 角丸を強調したカード(やわらかすぎる)
- ドロップシャドウの多用(浮きすぎる)
- 4 色以上の同時使用(色は厳格に管理)
- アニメーションの多用(動きが多すぎると安っぽい)
- 絵文字の使用
- カタカナビジネス用語の濫用

これらを排除することで、bizmote らしい独自性が立ち上がる。

## 2. カラーシステム

### 2.1 親ブランド: モノトーン

bizmote 全体の土台。コーポレートサイトのほとんどはこのパレットで構成する。

| 名称 | HEX | RGB | 用途 |
|---|---|---|---|
| **墨**(sumi) | `#1A1A1A` | rgb(26, 26, 26) | 主要テキスト、強調背景 |
| **濃灰**(noukai) | `#666666` | rgb(102, 102, 102) | 副次テキスト、ラベル |
| **中灰**(chukai) | `#999999` | rgb(153, 153, 153) | 三次テキスト、補助情報 |
| **淡灰**(tankai) | `#DDDDDD` | rgb(221, 221, 221) | 区切り線、ボーダー |
| **生成**(kinari) | `#FAFAF5` | rgb(250, 250, 245) | 通常背景(温かみのある白) |
| **純白** | `#FFFFFF` | rgb(255, 255, 255) | 限定的に使用(カード等) |

#### モノトーンの哲学

- **真っ白(#FFFFFF)を主背景にしない**: 冷たく無機質に見える
- **生成(#FAFAF5)を主背景にする**: 温かみと和の質感
- **真っ黒(#000000)も避ける**: 墨(#1A1A1A)で重みを軽減
- **これは紙の質感を意識した配色**: 印刷物としての美しさを Web に持ち込む

### 2.2 サブブランドカラー

3 事業それぞれの個性を表現する 3 色。各色は **意味と動きの方向性** を持つ。

#### bizFlag: 深紅(しんく)

| 階調 | HEX | 用途 |
|---|---|---|
| Deep | `#6B1A1A` | アクセント背景、強調 |
| Base | `#8B1E1E` | 主要色(リンク、ボタン) |
| Light | `#A83232` | ホバー、ハイライト |

**意味**: 規律と覚悟の赤
**動きの方向**: 下降(重力、浸透)
**インスピレーション**: 武将の旗印、漆器の朱、古文書の朱印
**避けるべき**: 朱色の明るさ、ピンク寄り、オレンジ寄り

#### bizShip: 蛍光青

| 階調 | HEX | 用途 |
|---|---|---|
| Deep | `#0095CC` | アクセント背景、強調 |
| Base | `#00C8FF` | 主要色(リンク、ボタン) |
| Light | `#5BD9FF` | ホバー、ハイライト、グロー |

**意味**: 挑戦と覚醒の青
**動きの方向**: 上昇(覚醒、発光)
**インスピレーション**: 夜明け前の空、ネオン、生体発光
**避けるべき**: 空色、ネイビー、紫寄り

#### bizLab: 蛍光緑

| 階調 | HEX | 用途 |
|---|---|---|
| Deep | `#00C060` | アクセント背景、強調 |
| Base | `#00FF7F` | 主要色(リンク、ボタン) |
| Light | `#5DFFAB` | ホバー、ハイライト、グロー |

**意味**: 生成と未来の緑
**動きの方向**: 放射(創出、増殖)
**インスピレーション**: ターミナル画面、AI 可視化、新芽
**避けるべき**: ライム、オリーブ、暖色寄り

### 2.3 ステートカラー(機能色)

機能的に必要なステート表現:

| 用途 | HEX | 備考 |
|---|---|---|
| **成功** | `#16A34A` | 限定使用 |
| **警告** | `#EAB308` | 限定使用 |
| **エラー** | `#DC2626` | フォームエラー等 |
| **情報** | `#0EA5E9` | 注釈等 |

これらはフォームのバリデーション等、機能的に必要な場面のみ使う。装飾には使わない。

### 2.4 配色のルール

#### ルール 1: 1 ページに 1 アクセントカラー

コーポレートトップは基本モノトーン。3 事業セクションでのみ各サブカラーが順次登場する。
各事業 LP(/flag/, /ship/, /lab/)では、その事業のサブカラーのみを使用。複数のサブカラーを同時に使わない。

#### ルール 2: アクセントカラーは画面の 5-10% に抑える

サブカラーが画面を支配しないこと。**主役はモノトーン、アクセントが効くから印象に残る**。

#### ルール 3: 文字に色を多用しない

リンク・強調以外は、本文は墨(#1A1A1A)。サブカラーで本文を書かない。

#### ルール 4: 背景色は 3 種類まで

1 ページ内で背景色を増やしすぎない:
- 主背景: 生成(#FAFAF5)
- サブセクション背景: 純白(#FFFFFF) または 墨(#1A1A1A)
- 限定的なアクセント背景: サブカラーの Deep 階調

## 3. タイポグラフィシステム

### 3.1 フォントファミリー

bizmote は **3 種類のフォント** を厳格に使い分ける。

#### 日本語見出し: Shippori Mincho(しっぽり明朝)

- **Google Fonts**: https://fonts.google.com/specimen/Shippori+Mincho
- **使用箇所**: 大見出し、タグライン、引用
- **ウェイト**: 200(極細)、400(レギュラー)、500(中)、700(太)
- **特徴**: 上品で文芸的、bizmote の哲学性を表現
- **CSS 例**:
  ```css
  font-family: 'Shippori Mincho', serif;
  font-weight: 200;
  letter-spacing: -0.02em;
  ```

#### 日本語本文: Noto Serif JP

- **Google Fonts**: https://fonts.google.com/noto/specimen/Noto+Serif+JP
- **使用箇所**: 本文、説明文、記事
- **ウェイト**: 400(レギュラー)、500(中)、700(太)
- **特徴**: 読みやすく、Shippori Mincho と相性が良い
- **CSS 例**:
  ```css
  font-family: 'Noto Serif JP', serif;
  font-weight: 400;
  line-height: 1.9;
  ```

#### 英語見出し・本文: Cormorant Garamond

- **Google Fonts**: https://fonts.google.com/specimen/Cormorant+Garamond
- **使用箇所**: 英語見出し、英語本文
- **ウェイト**: 300(細)、400(レギュラー)、500(中)、700(太)
- **特徴**: クラシカルで上品、Shippori Mincho と並んで違和感がない
- **CSS 例**:
  ```css
  font-family: 'Cormorant Garamond', serif;
  font-weight: 300;
  letter-spacing: 0.01em;
  ```

#### ラベル・コード・メタデータ: JetBrains Mono

- **Google Fonts**: https://fonts.google.com/specimen/JetBrains+Mono
- **使用箇所**: ラベル(BIZMOTE — GTM ARCHITECTURE FIRM 等)、コード、ナビゲーション、メタデータ
- **ウェイト**: 300(細)、400(レギュラー)、500(中)
- **特徴**: テクニカル感、英大文字+wide letter-spacing で技術的洗練を表現
- **CSS 例**:
  ```css
  font-family: 'JetBrains Mono', monospace;
  font-weight: 400;
  font-size: 11px;
  letter-spacing: 0.3em;
  text-transform: uppercase;
  ```

### 3.2 絶対に使わないフォント

以下は bizmote では使用禁止:

- **Arial / Helvetica**(汎用すぎる、無個性)
- **Inter**(AI 時代の SaaS スタートアップに使われすぎ)
- **Roboto**(同上)
- **Times New Roman**(古臭く、Web に向かない)
- **Comic Sans 等のカジュアルフォント**(論外)
- **システムフォント default**(個性がない)

### 3.4 タイポグラフィの実装ルール

#### ルール 1: letter-spacing(文字間)の使い分け

- **巨大見出し**: -0.02em(やや詰める、印象的に)
- **本文**: 0(調整しない)
- **ラベル(英大文字)**: 0.2-0.3em(広く取る、テクニカル感)

#### ルール 2: line-height(行間)の使い分け

- **見出し**: 1.05-1.4(タイトに)
- **本文**: 1.9(広めに、読みやすさ重視)
- **ラベル**: 1.5

#### ルール 3: font-weight(太さ)の使い分け

bizmote は **細いウェイト(thin/light)を多用** する。これが上品さの源。

- **見出しの基本**: 200-300(極細)
- **本文の基本**: 400(レギュラー)
- **強調**: 500(中)
- **太字は最小限**: 700 はほぼ使わない

#### ルール 4: 引用符の使い方

- **日本語**: 「」(かぎ括弧)を基本
- **強調**: " "(ダブルクォート)
- **絶対に使わない**: 『』(二重かぎ括弧)、‘ ’(シングルクォート)

## 4. 余白システム

### 4.1 8pt グリッド

すべての余白は **8 の倍数** で構成する。これにより、画面全体に統一感とリズムが生まれる。

#### スペーシングスケール

| 名称 | 値 | 用途 |
|---|---|---|
| **xs** | 4px | 最小余白(アイコンとテキストの間等) |
| **sm** | 8px | 小さい余白 |
| **md** | 16px | 標準余白 |
| **lg** | 24px | やや広い余白 |
| **xl** | 32px | セクション内の区切り |
| **2xl** | 48px | サブセクション間 |
| **3xl** | 64px | セクション内余白 |
| **4xl** | 96px | セクション間 |
| **5xl** | 128px | 大きなセクション間 |
| **6xl** | 192px | ヒーロー級の余白 |

### 4.2 余白の設計原則

#### 原則 1: 余白を恐れない

bizmote のデザインは **余白で品格を作る**。「もう少し詰めたい」と思っても、むしろ広げる。

#### 原則 2: 縦の余白を広く取る

セクション間の縦余白は、**最低でも 96px(4xl)、可能なら 128-192px**。これが文芸性を生む。

#### 原則 3: 左右の余白を取る

コンテンツの最大幅:
- **本文系**: 720px(読みやすさ最優先)
- **見出し系**: 1200-1440px(画面いっぱいに使う)
- **画面の左右**: 最低 80px(デスクトップ)/ 32px(モバイル)

#### 原則 4: 行間を広く

本文の行間は 1.9。これは一般的な Web サイトより広め。**読み物として扱う** ための設計。

### 4.3 グリッドシステム

レイアウトの基本グリッド:

#### デスクトップ

- **カラム数**: 12
- **ガター**: 24px
- **マージン**: 80px(画面端からの余白)

#### モバイル

- **カラム数**: 4
- **ガター**: 16px
- **マージン**: 32px

## 6. アニメーション仕様

### 6.1 アニメーションの哲学

bizmote のアニメーションは **静謐(せいひつ)** を最優先する。

- **遅い動きが上品**: 0.6 秒以下の速い動きは安っぽく見える
- **少ない動きが効く**: ページ全体で 5-7 個程度のアニメーションが適切
- **イージングが命**: linear や ease-out ではなく、こだわりのカーブを使う

### 6.2 デュレーション(動きの長さ)

| 用途 | デュレーション | 備考 |
|---|---|---|
| **ホバー(ボタン等)** | 200-300ms | 反応の良さを保つ |
| **フェードイン** | 800-1200ms | ゆっくり、瞑想的に |
| **スクロール連動** | スクロール量に依存 | スクラビング |
| **ページ遷移** | 600-1000ms | 急ぎすぎない |
| **動画ループ** | 15 秒 | ヒーロー動画の長さ |

### 6.3 イージング(動きのカーブ)

CSS で使うイージング関数:

| 用途 | CSS | 備考 |
|---|---|---|
| **デフォルト** | `cubic-bezier(0.4, 0, 0.2, 1)` | Material Design の Standard |
| **入場(現れる)** | `cubic-bezier(0, 0, 0.2, 1)` | Decelerate |
| **退場(消える)** | `cubic-bezier(0.4, 0, 1, 1)` | Accelerate |
| **柔らかい動き** | `cubic-bezier(0.34, 1.56, 0.64, 1)` | Spring back(限定使用) |

### 6.4 主要アニメーションパターン

#### パターン 1: スクロールフェードイン

要素が画面に入ったときにフェードインする:

```
[初期状態] opacity: 0; transform: translateY(40px);
   ↓ (要素が viewport に 20% 入った瞬間)
[完了] opacity: 1; transform: translateY(0);

duration: 1000ms
easing: cubic-bezier(0, 0, 0.2, 1)
```

GSAP + ScrollTrigger で実装。

#### パターン 2: スタガード(順次)出現

複数要素が順番に現れる:

```
要素 1: delay 0ms
要素 2: delay 100ms
要素 3: delay 200ms
要素 4: delay 300ms
```

カード一覧やリスト表示で使う。

#### パターン 3: テキスト分割アニメーション

タイトルが文字単位で現れる(限定使用):

```
[次] [の] [常] [識] [を] [つ] [く] [る] [。]
  ↑
  各文字が 100ms ずつ遅れて fade in + translate up
```

ヒーローセクションのみ。多用しない。

#### パターン 4: スクロール連動の動画再生

ScrollTrigger で動画の現在再生位置をスクロール量に同期:

```
[scrollProgress] 0% → 動画 0 秒
[scrollProgress] 50% → 動画 7.5 秒
[scrollProgress] 100% → 動画 15 秒
```

セクションが画面を通過する間、動画がスクロールに連動。

#### パターン 5: セクション間の動画クロスフェード

セクションが切り替わるとき、背景動画が静かに切り替わる:

```
[セクション 1: コーポレート墨絵動画]
  ↓ scroll
[クロスフェード 1.5 秒]
  ↓
[セクション 3: bizFlag 深紅動画]
```

### 6.5 アクセシビリティ配慮

#### prefers-reduced-motion 対応

ユーザーが OS で「動きを減らす」設定をしている場合:

```css
@media (prefers-reduced-motion: reduce) {
  /* すべてのアニメーションを無効化 or 最小化 */
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }

  /* 動画は静止画にフォールバック */
  video {
    display: none;
  }
}
```

これは **必須実装**。アクセシビリティ違反になる。

## 8. 各事業ブランドの差別化

### 8.1 サマリー表

各事業 LP を訪れた時、**色だけでなく動き・トーン・配置で違いを感じる** ようにする。

| 要素 | bizmote | bizFlag | bizShip | bizLab |
|---|---|---|---|---|
| **色** | モノトーン | 深紅 | 蛍光青 | 蛍光緑 |
| **動きの方向** | 横の流れ | 下降 | 上昇 | 放射 |
| **トーン** | 哲学的・中立 | 規律・重厚 | 覚醒・上昇 | 生成・知性 |
| **対象感情** | 信頼・哲学 | 覚悟・実行力 | 希望・挑戦 | 興奮・未来 |
| **タイポグラフィの強度** | 静か | 力強い | 軽やか | 鋭い |

### 8.2 各 LP の固有スタイル

#### bizFlag LP の特徴

- **背景動画**: 深紅の墨絵が下方向に流れる
- **本文の余白**: bizmote 全体より少し広めに(品格)
- **ボタン**: 深紅の Base 色がアクセント
- **フォントウェイト**: 中(500)を多用、力強さを表現

#### bizShip LP の特徴

- **背景動画**: 蛍光青の光が下から上へ
- **本文の余白**: 標準
- **ボタン**: 蛍光青がアクセント、グロー効果あり
- **フォントウェイト**: 細(200-300)を多用、軽やかさを表現

#### bizLab LP の特徴

- **背景動画**: 蛍光緑の粒子が放射状に
- **本文の余白**: 少しタイト(情報密度高め)
- **ボタン**: 蛍光緑がアクセント
- **背景に微細グリッド**: 5% 透過のグリッドパターン
- **フォントウェイト**: 中-太(500-700)も使用、技術感

## 9. レスポンシブデザイン

### 9.1 ブレイクポイント

| 名称 | 幅 | 対象 |
|---|---|---|
| **mobile** | 〜639px | スマホ |
| **tablet** | 640〜1023px | タブレット |
| **desktop** | 1024〜1439px | ノート PC |
| **wide** | 1440px〜 | デスクトップ |

### 9.2 レスポンシブの設計原則

#### 原則 1: モバイルファースト

すべての CSS はモバイル基準で書き、デスクトップで拡張する。

#### 原則 2: 巨大見出しは大胆にスケール

- **デスクトップ**: 120px
- **モバイル**: 56-64px

ただし、**モバイルでも文字が大きく印象的であること**。萎縮しすぎない。

#### 原則 3: 余白も縮小する

- **デスクトップ**: セクション間 192px
- **モバイル**: セクション間 96px

ただし、詰めすぎないこと。

#### 原則 4: 動画はモバイルでも再生

ただし、容量を抑えた軽量版を別途用意:
- **デスクトップ**: 1920×1080、5MB
- **モバイル**: 720×1280、2MB

データ通信制限の人向けに、静止画フォールバックも実装。

## 10. ダークモード(将来検討)

ダークモードは **ローンチ時には実装しない**。理由:

- bizmote のブランドは「生成色(#FAFAF5)+ 墨(#1A1A1A)」が基調で、ダークモードを別途デザインすると2倍のデザインコストがかかる
- ヒーローやサブブランドセクションはすでに暗い背景を使っているため、視覚的にダークモードに近い体験を提供している
- 将来必要になった時点で別途検討
